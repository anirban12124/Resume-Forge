1. Introduction
1.1 Purpose
This Software Requirements Specification defines the complete functional, non-functional, and architectural requirements for ResumeForge (working title), a web application that helps developers generate ATS-optimized resumes tailored to specific job descriptions. The system ingests a user's professional data (resume, GitHub projects, internships), stores it in a structured knowledge vault, and uses LLM-powered pipelines to forge targeted resumes in LaTeX format.
1.2 Scope
The application is designed for single-user/personal use with the architecture capable of supporting multiple users. It covers:

User authentication (GitHub OAuth, Google OAuth, email/password)
Knowledge vault creation and management (max 3 per user)
PDF/LaTeX resume ingestion and structured data extraction
GitHub repository analysis and metadata extraction
LLM-powered resume generation targeting specific JDs
LaTeX compilation and PDF delivery
Automated job scraping from 4 Indian job platforms
Resume archive with view, download, and LaTeX source access

1.3 Intended Audience
Developer (self), future contributors, and portfolio reviewers.
